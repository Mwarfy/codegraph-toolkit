# @liby-tools/codegraph-mcp

MCP server exposing codegraph snapshot queries — architectural intelligence for Claude Code.

## Why

LSP gives semantic fine-grained intelligence (symbols, types, refs).
codegraph-mcp gives **structural coarse-grained** intelligence: files, ADRs,
truth-points, cycles, technical debt. Together: complete architectural
context, push-and-pull, no redundancy.

Without codegraph-mcp, the codegraph snapshot is a static artifact only
consumed by hooks (push). With it, queries become callable on-demand
(pull) — the same way LSP exposes semantic queries on-demand.

## Tools

| Tool | Use case |
|---|---|
| `codegraph_context(file_path)` | Get the architectural context of a file: in/out degree, top importers, exports problématiques, cycles, truth-points, long functions, magic numbers, test coverage. Same info as the PostToolUse hook, but on-demand. Use BEFORE editing. |
| `codegraph_who_imports(file_path, include_indirect?)` | List every file that imports the given file (FILE-level). Distinct from `lsp_find_references` which is SYMBOL-level. For impact analysis at the module boundary. |
| `codegraph_truth_point_for(file_path)` | Detail the file's participation in truth-points: which canonical concepts it writes / reads / mirrors. Critical before modifying anything DB/SSOT-related. |
| `codegraph_recent(file_path, weeks?)` | Git archaeology: commits in last N weeks, top contributor, file age. Understand why a file was recently touched. |
| `codegraph_uncovered(critical_only?, limit?)` | List source files without test coverage, ranked by criticality (truth-point writers + hubs first). Find test gaps that actually matter. |

## Setup

### Prerequisites

- **Node.js** ≥ 18
- A consuming project that has run `codegraph analyze` at least once (creates `.codegraph/snapshot-*.json`)

### Install

```bash
npm install -g @liby-tools/codegraph-mcp
# OR if installed via this monorepo:
npm link --workspace=@liby-tools/codegraph-mcp
```

### Wire in Claude Code

Add to your project's `.mcp.json`:

```json
{
  "mcpServers": {
    "codegraph": {
      "command": "codegraph-mcp",
      "cwd": "/absolute/path/to/your/project"
    }
  }
}
```

Restart Claude Code. The 5 tools become available as `mcp__codegraph__*`.

### Verify

In Claude Code:
> `codegraph_context(file_path: "path/to/some/file.ts")`

Should return the architectural context block.

## How it works

The server reads the latest `.codegraph/snapshot-*.json` from `<cwd>/.codegraph/`. Snapshot is mtime-cached in RAM — no re-parse on each tool call, refresh only when snapshot file changes (regenerated by post-commit hook).

100% local. No network. No telemetry. ~300 LOC total.

## Architecture

```
┌────────────────────────┐
│  Claude Code (frame)   │
└─────┬─────────────┬────┘
      │ stdio       │ stdio
      ▼             ▼
   MCP "lsp"    MCP "codegraph"
   semantic     architectural
   ───────────  ──────────────
   • hover      • context
   • find_refs  • who_imports
   • diagnostics• truth_point_for
   • rename     • recent
   ...          • uncovered
```

LSP and codegraph-mcp are complementary, not redundant. LSP zooms on symbols, codegraph-mcp zooms on file-level architecture. Use both for full picture.
